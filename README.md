# LLM_Semantic_Comparison
# 大语言模型中文语义理解能力横向对比测试报告

> **项目说明**：本项目通过阿里云 PAI-DSW 平台，在 32GB CPU 环境下实现了三款主流 7B 级国产大语言模型的本地化部署，并完成了深度语义对比测试。

---

## 1. 项目简介
本项目旨在评估不同大语言模型（LLM）对中文复杂语境的理解深度。通过五个维度的极端语义测试，横向对比各模型在歧义识别、逻辑推理及社交语用方面的差异。

### 部署模型清单：
*   **ChatGLM3-6B**: 智谱AI出品，国产 LLM 的先驱。
*   **Qwen-7B-Chat**: 阿里巴巴通义千问，以强悍的中文理解著称。
*   **DeepSeek-LLM-7B-Chat**: 深度求索出品，逻辑推理与工程优化标杆。

---

## 2. 环境配置与优化
由于硬件限制为 **CPU-only**，本项目采用了针对性的技术方案以平衡运行效率与稳定性。

*   **硬件规格**：阿里云 DSW 实例 / CPU 核心 / 32GB RAM。
*   **技术架构**：
    *   **容器环境**：Python 3.10 + Conda 虚拟环境。
    *   **存储格式**：统一采用 `bfloat16` 精度加载，单模型内存占用约 **14.5GB**，完美适配 32GB 内存容量。
    *   **工程补丁**：通过代码干预修复了旧版 `transformers` 库与新版模型代码之间的兼容性冲突。

## 5. 横向对比分析 (Core Analysis)

### 5.1 语义解构与文化对齐深度
*   **Qwen-7B-Chat** 展示了最强的“中式语感”。在处理诸如“意思”这种高度依赖情境的词汇时，Qwen 能准确识别潜台词。
*   **ChatGLM3-6B** 虽然在通用场景表现稳健，但在极端语境下容易产生语域混淆（中英混杂）。

### 5.2 逻辑建模与递归推理稳定性
*   **DeepSeek-7B** 在处理多层嵌套逻辑时，逻辑一致性最高。这证明其在预训练阶段对高质量逻辑数据的吸收效率极佳，更适合作为逻辑链推理任务的底层。

### 5.3 CPU 环境下的工程适配表现
*   **资源占用**：32GB RAM 是 CPU 部署 7B 模型的“甜点区”。使用 `low_cpu_mem_usage=False` 可有效避开 Jupyter 环境下的加载死锁。
*   **兼容性**：新一代模型（Qwen/DeepSeek）的工程架构明显更优，对现代库的适配性高于早期框架模型。

---

## 6. 实验结论
综合评估，**Qwen-7B-Chat** 是处理复杂中文理解任务的首选方案，其文化对齐深度具有代际优势；**DeepSeek-7B** 则在逻辑推理稳定性及工程化友好度上表现突出。
